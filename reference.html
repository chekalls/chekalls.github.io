Référence - Gestion des profils

Page de documentation statique présentant les écrans et flux pour la gestion des profils côté client (utilisateur) et côté administrateur/modérateur. Conçue comme un guide de référence ou maquette côté frontend.

Utilisateur (propriétaire de profil)

Écran d'édition

  • Formulaire affichant les champs du profil :
    • Nom complet (non modifiable)
    • Bio, formation, expériences, compétences, spécialités, contacts (téléphone, LinkedIn, etc.)
  • Pour chaque champ, une case à cocher "rendre public" ou "privé".
  • Bouton Sauvegarder en bas, qui déclenche un PUT /profiles/{id}.
  • Bouton Prévisualiser qui affiche la version publique (flux GET /profiles/{id}).
  • Validation serveur : longueurs, formats, pas de HTML.
  • En cas d'erreur, le même écran réaffiche le formulaire avec messages d'erreur en rouge.

Écran de visualisation publique

Affiche uniquement les champs marqués publics. Les champs privés sont masqués ou remplacés par "(privé)".

Inclut un lien "Modifier mon profil" visible uniquement pour le propriétaire.

Administrateur / Modérateur

Vue de modération

  • Panel de recherche (annuaire) qui ouvre les profils publics.
  • Lorsqu'un profil est ouvert, boutons supplémentaires :
    • Suspendre / Bannir l'utilisateur (envoie un signal à POST /users/{id}/suspend ou similaire).
    • Modifier : accès à un formulaire similaire à celui du propriétaire, mais pouvant éditer tous les champs.
  • Modification entraîne création d'une entrée d'audit (modérateur, champs modifiés, timestamp).
